# SO-ARM100 MuJoCo Model-Based Manipulation

基于 MuJoCo 物理引擎的 SO-ARM100 6-DOF 机械臂仿真系统，集成了正运动学 (FK)、逆运动学 (IK)、三次多项式轨迹生成与 PD 控制执行的完整 model-based manipulation pipeline。

---

## 项目结构

```
mujoco_sim/
├── main.py                          # 主程序入口 (5个Demo)
├── simulator.py                     # MuJoCo 仿真器 (模型加载/PD控制/传感器读取)
├── trajectory.py                    # 三次多项式轨迹生成 (关节空间 + 笛卡尔空间)
├── requirements.txt                 # Python 依赖
├── __init__.py
├── models/
│   └── so_arm100.xml                # SO-ARM100 MJCF 模型文件
├── kinematics/
│   ├── __init__.py
│   ├── forward_kinematics.py        # 正运动学 (DH变换矩阵链)
│   └── inverse_kinematics.py        # 逆运动学 (解析/数值/神经网络)
└── utils/
    ├── __init__.py
    └── data_generator.py            # IK 神经网络训练数据生成
```

## 模块详解

### 1. MJCF 机器人模型 (`models/so_arm100.xml`)

基于 SO-ARM100 硬件参数构建的 MuJoCo MJCF 模型：

| 参数 | 值 |
|---|---|
| 自由度 | 6 (全部 revolute) |
| 旋转轴序列 | Z, X, X, X, Y, X |
| 仿真步长 | 0.002s (500Hz) |
| 积分器 | implicit (刚性稳定) |

**关节定义与限位：**

| 关节 | 旋转轴 | 范围 | 连杆偏移 (mm) |
|---|---|---|---|
| Joint 1 (Base) | Z | ±90° | [0, 0, 0] |
| Joint 2 (Shoulder) | X | 0°~180° | [0, 31.15, 119.7] |
| Joint 3 (Elbow) | X | -170°~25° | [0, 112.35, -28.0] |
| Joint 4 (Wrist 1) | X | ±180° | [0, 134.9, 4.85] |
| Joint 5 (Wrist 2) | Y | ±90° | [0, 54.8, 0] |
| Joint 6 (Wrist 3) | X | ±180° | [0, 31.5, 20.0] |
| End-Effector | — | — | [0, 76.0, -11.5] |

传感器配置：
- 6 × 关节位置传感器 (`jointpos`)
- 6 × 关节速度传感器 (`jointvel`)
- 末端执行器位置 (`framepos`)
- 末端执行器姿态 (`framequat`)

### 2. 正运动学 (`kinematics/forward_kinematics.py`)

通过齐次变换矩阵链计算关节角到末端执行器位姿的映射：

$$T_{ee} = \prod_{i=1}^{6} T_i(\theta_i) \cdot T_{grab}$$

其中每个 $T_i$ 根据旋转轴 (X/Y/Z) 构建：

$$T_x(\theta) = \begin{bmatrix} 1 & 0 & 0 & d_x \\ 0 & \cos\theta & -\sin\theta & d_y \\ 0 & \sin\theta & \cos\theta & d_z \\ 0 & 0 & 0 & 1 \end{bmatrix}$$

**API：**
```python
from mujoco_sim.kinematics import ForwardKinematics

fk = ForwardKinematics()
quaternion, position = fk.compute([0.3, 1.0, -1.2, -0.5, 0.2, 0.1])
# quaternion: [x, y, z, w], position: [x, y, z] (mm)
```

### 3. 逆运动学 (`kinematics/inverse_kinematics.py`)

提供三种 IK 求解策略：

#### 3.1 解析法 (`analytical`)

利用机械臂几何结构分解为：
1. **腕部中心定位**：从目标位姿反推腕部中心 $p_w = p_{ee} - R_{target} \cdot d_{wrist}$
2. **基座旋转** (J1)：$\theta_1 = \text{atan2}(p_{w,y}, p_{w,x})$
3. **平面 2-link IK** (J2, J3)：余弦定理求解肘关节
4. **腕部姿态** (J4, J5, J6)：从剩余旋转矩阵提取 X-Y-X 欧拉角

#### 3.2 数值法 (`numerical`)

阻尼最小二乘法 (Levenberg-Marquardt)：

$$\Delta\theta = J^T (JJ^T + \lambda I)^{-1} \cdot e$$

其中 $e = [e_{pos}; e_{orient}]$ 为 6D 误差向量，$J$ 为 6×6 雅可比矩阵（数值差分计算）。

#### 3.3 神经网络法 (`neural_network`)

加载预训练 Keras 模型（架构：128→96→48→6），输入 13 维特征向量：
- 6 个参考关节角
- 4 个目标四元数分量
- 3 个目标位置坐标

**API：**
```python
from mujoco_sim.kinematics import InverseKinematics

ik = InverseKinematics()

# 数值法 IK
angles = ik.numerical(target_quat, target_pos)

# 解析法 IK
angles = ik.analytical(target_quat, target_pos)

# 神经网络 IK (需预加载模型)
ik.load_nn_model("model.keras", "scalers.pkl")
angles = ik.neural_network(target_quat, target_pos, reference_angles=current)
```

### 4. 轨迹生成 (`trajectory.py`)

#### 三次多项式插值

给定边界条件 $(q_0, \dot{q}_0, q_f, \dot{q}_f, T)$，计算多项式系数：

$$q(t) = a_0 + a_1 t + a_2 t^2 + a_3 t^3$$

$$\begin{cases} a_0 = q_0 \\ a_1 = \dot{q}_0 \\ a_2 = \frac{3(q_f - q_0)}{T^2} - \frac{2\dot{q}_0}{T} - \frac{\dot{q}_f}{T} \\ a_3 = \frac{2(q_0 - q_f)}{T^3} + \frac{\dot{q}_0 + \dot{q}_f}{T^2} \end{cases}$$

**支持模式：**
- **关节空间轨迹**：直接在各关节空间插值
- **笛卡尔空间轨迹**：位置 + 欧拉角插值，经 IK 转换为关节轨迹
- **多路径点轨迹**：将多段轨迹自动串联

**API：**
```python
from mujoco_sim.trajectory import TrajectoryGenerator

tg = TrajectoryGenerator()

# 关节空间轨迹
traj = tg.joint_space_trajectory(start_angles, end_angles, duration=3.0, frequency=50)

# 笛卡尔空间轨迹
task_traj = tg.task_space_trajectory(start_pos, end_pos, start_euler, end_euler, ...)
joint_traj = tg.task_to_joint_trajectory(task_traj, method="numerical")

# 多路径点
full_traj = tg.multi_waypoint_joint_trajectory(waypoints, durations, frequency=50)
```

### 5. MuJoCo 仿真器 (`simulator.py`)

封装 MuJoCo C API，提供高层接口：

| 方法 | 功能 |
|---|---|
| `reset(angles)` | 重置仿真，可设初始关节角 |
| `step(n)` | 前进 n 个仿真步 |
| `set_joint_targets_pd(targets, kp, kd)` | PD 位置控制：$\tau = k_p(q_d - q) - k_d \dot{q}$ |
| `execute_trajectory(traj, freq)` | 执行完整轨迹（含实时渲染） |
| `get_joint_positions()` | 读取关节位置传感器 |
| `get_ee_position()` | 读取末端执行器位置 |
| `launch_viewer()` | 启动交互式 3D 查看器 |

**API：**
```python
from mujoco_sim.simulator import MuJoCoSimulator

sim = MuJoCoSimulator(render=True)
sim.reset([0, 0.8, -1.5, -1.0, 0, 0])
states = sim.execute_trajectory(trajectory, frequency=50, record=True)
sim.close()
```

### 6. IK 数据集生成 (`utils/data_generator.py`)

为神经网络 IK 训练生成数据集：
- 在关节限位内均匀采样参考角度
- 对每组参考角度施加 ±2° 的随机扰动
- 通过 FK 计算目标位姿
- 输出 CSV：13 个输入特征 + 6 个目标关节角

```python
from mujoco_sim.utils import generate_ik_dataset

df = generate_ik_dataset(num_samples=5000, perturbation_deg=2.0, perturbation_iters=8)
# 生成 40,000 条训练样本 → ik_dataset.csv
```

## Demo 说明

### Demo 1: FK 验证 (`--demo fk`)
将自定义 FK 计算结果与 MuJoCo 内部运动学对比，验证 MJCF 模型与 DH 参数的一致性。

### Demo 2: IK 求解 (`--demo ik`)
对同一目标位姿分别使用解析法和数值法 IK 求解，对比精度和效果。启用 `--render` 可看到机械臂运动到目标位姿。

### Demo 3: 轨迹执行 (`--demo traj`)
生成关节空间三次多项式轨迹，验证轨迹平滑性（速度/加速度连续），并在仿真器中执行。

### Demo 4: 抓取放置 (`--demo pick`)
模拟完整的 pick-and-place 流程：

```
Home → Pre-pick → Pick ↓ → Lift ↑ → Transport → → Place ↓ → Retract ↑ → Home
```

8 个路径点通过三次多项式平滑串联，总时长 14 秒。

### Demo 5: 笛卡尔轨迹 (`--demo task`)
在笛卡尔空间定义起止位姿（位置 + 姿态），通过三次多项式插值生成任务空间轨迹，再经数值 IK 逐点转换为关节空间轨迹执行。

## 从原项目迁移说明

| 原模块 (ROS + Unity Simulator) | 新模块 (MuJoCo) | 变更 |
|---|---|---|
| `hw1/SOARM101.py` | `kinematics/forward_kinematics.py` | 保留相同 DH 参数和变换矩阵；增加 quaternion/rotation matrix 互转 |
| `hw2/SOARM101.py` | `kinematics/inverse_kinematics.py` | 保留解析 IK + 数值 IK (Jacobian damped LS)；移除 ROS 消息类型依赖 |
| `hw2/cubic_trajectory_generation.py` | `trajectory.py` | 保留三次多项式算法；移除 ROS Node / Publisher / Subscriber |
| `hw3/ik_nn.py` + `data_gen.py` | `utils/data_generator.py` + IK `neural_network()` | 保留 NN 架构和数据流；模型加载与推理集成到 IK 类 |
| Unity CUTeR Arm Simulator | `simulator.py` + `models/so_arm100.xml` | 仿真引擎从 Unity 切换为 MuJoCo；PD 控制替代 ROS topic 通信 |

**关键改进：**
1. **去除 ROS 依赖**：所有通信通过 Python 函数调用，零配置即可运行
2. **物理仿真**：MuJoCo 提供真实的刚体动力学、碰撞检测、重力模拟
3. **统一接口**：FK / IK / 轨迹 / 仿真通过 Python API 无缝集成
4. **可视化**：MuJoCo 内建 3D 查看器，支持交互式相机控制
