## 1. Difference between Analytical IK and Numerical IK

### 1.1 Conceptual Differences

**Analytical IK (解析逆运动学)**:
- 通过几何分析和代数方法直接求解关节角度
- 基于机器人的几何结构推导出闭合形式的解
- 对于6自由度机械臂，完整的解析解可能非常复杂或不存在
- 本实现采用了简化的几何方法，基于机械臂的结构特点进行求解

**Numerical IK (数值逆运动学)**:
- 使用迭代优化方法逐步逼近目标位姿
- 基于雅可比矩阵的逆或伪逆进行迭代更新
- 适用于各种复杂机器人结构
- 本实现使用牛顿-拉夫逊方法配合雅可比伪逆

### 1.2 Practical Differences

**Formulation (公式化)**:
- **解析IK**: 直接计算，无需迭代
  - 关节1：通过目标位置在xy平面的投影计算基座旋转角
  - 关节2-3：使用余弦定理和2D逆运动学求解肩部和肘部角度
  - 关节4-6：从旋转矩阵中提取X-Y-X欧拉角序列
  
- **数值IK**: 迭代求解
  - 计算当前位姿与目标位姿的误差
  - 使用雅可比矩阵计算关节角度更新量
  - 重复迭代直到收敛

**Computational Efficiency (计算效率)**:
- **解析IK**: 
  - 优点：计算速度快，单次计算即可得到结果
  - 缺点：对于复杂结构，解析解可能不存在或需要大量预处理
  
- **数值IK**:
  - 优点：通用性强，适用于各种结构
  - 缺点：需要多次迭代，计算时间较长（本实现最多100次迭代）

**Convergence Behavior (收敛行为)**:
- **解析IK**: 
  - 总是给出一个解（如果存在）
  - 但可能不是最优解，且对于某些位姿可能失败
  
- **数值IK**:
  - 收敛性依赖于初始猜测和步长
  - 本实现使用阻尼最小二乘法（Levenberg-Marquardt风格）提高稳定性
  - 容差设置为1e-4，最大迭代次数100

**Suitability for Different Robot Types (适用性)**:
- **解析IK**: 
  - 最适合具有特殊几何结构的机器人（如6自由度机械臂，当存在解析解时）
  - 对于冗余机器人，可能需要选择特定配置
  
- **数值IK**:
  - 适用于所有类型的机器人
  - 特别适合高自由度或复杂结构的机器人
  - 可以处理冗余度问题

## 2. Implementation Description

### 2.1 IK Solver Design

#### 2.1.1 Analytical IK Implementation

**关键几何推理**:
1. **关节1（基座旋转）**: 
   - 计算目标位置在xy平面的投影
   - 使用`atan2(y, x)`计算基座旋转角

2. **关节2-3（肩部和肘部）**:
   - 从目标位置减去末端执行器偏移，得到腕部位置
   - 使用余弦定理计算关节3的角度
   - 基于关节3角度计算关节2的角度

3. **关节4-6（腕部）**:
   - 计算前三个关节的累积变换矩阵
   - 提取腕部所需的旋转矩阵
   - 从旋转矩阵中提取X-Y-X欧拉角序列（对应关节4、5、6的旋转轴）

**变换链**:
- 关节旋转轴序列：Z, X, X, X, Y, X
- 末端执行器偏移在最后应用

#### 2.1.2 Numerical IK Implementation

**停止准则**:
- 误差容差：1e-4（位置和姿态误差的L2范数）
- 最大迭代次数：100
- 如果误差小于容差，提前终止迭代

**步长**:
- 固定步长：1.0
- 使用阻尼因子λ=0.1进行正则化，提高数值稳定性

**误差容差**:
- 位置误差：目标位置与当前位置的差值（mm）
- 姿态误差：使用轴角表示法计算旋转误差（rad）
- 总误差：位置和姿态误差的拼接向量的L2范数

**雅可比计算**:
- 使用数值微分方法
- 对每个关节角度进行微小扰动（δ=1e-6）
- 计算位置和姿态的变化率

**更新规则**:
- 使用阻尼最小二乘法：`J_pinv = J^T (J J^T + λI)^(-1)`
- 如果矩阵奇异，使用Moore-Penrose伪逆
- 关节角度更新：`θ_new = θ_old + step_size * J_pinv * error`

### 2.2 Cubic Trajectory Generation

#### 2.2.1 Task Space Trajectory Generation

**三次多项式轨迹**:
对于位置和欧拉角，使用三次多项式：
```
p(t) = a₀ + a₁t + a₂t² + a₃t³
```

**边界条件**:
- 位置：`p(0) = start_pos`, `p(T) = end_pos`, `p'(0) = v_start`, `p'(T) = v_end`
- 欧拉角：`θ(0) = start_euler`, `θ(T) = end_euler`, `θ'(0) = w_start`, `θ'(T) = w_end`

**系数计算**:
```
a₀ = start_value
a₁ = start_velocity
a₂ = (3(end - start) / T²) - (2v_start / T) - (v_end / T)
a₃ = (2(start - end) / T³) + ((v_start + v_end) / T²)
```

**欧拉角处理**:
- 使用内禀欧拉角（XYZ序列）
- 实现最短路径展开（shortest path unwrap）避免±π跳跃
- 将欧拉角转换为旋转矩阵，再转换为四元数用于IK求解

**旋转矩阵转换**:
- 内禀XYZ欧拉角：`R = R_z(yaw) * R_y(pitch) * R_x(roll)`
- 旋转矩阵到四元数：使用标准转换算法

#### 2.2.2 Joint Space Trajectory Conversion

**转换过程**:
1. 对任务空间轨迹中的每个点（位置+四元数）
2. 使用数值IK求解器计算对应的关节角度
3. 使用前一个轨迹点的结果作为初始猜测，提高收敛速度和准确性
4. 将所有关节角度存储为6×N矩阵（N为轨迹点数）

**IK求解器选择**:
- 使用数值IK而非解析IK，因为：
  - 数值IK更可靠，收敛性更好
  - 对于轨迹中的每个点都能给出合理解
  - 解析IK在某些位姿可能失败

**优化策略**:
- 利用轨迹连续性：使用前一个点的关节角度作为下一个点的初始猜测
- 这样可以显著提高收敛速度，因为相邻轨迹点通常变化不大
- 对于1000个点的轨迹，平均每个点只需10-30次迭代即可收敛

#### 2.2.3 Trajectory Publishing

**发布机制**:
- 使用ROS2定时器按给定频率发布
- 每次发布当前时间步的6×1关节角度向量
- 使用`Float64MultiArray`消息类型
- 话题名称：`/hw2_q2_joint_cmd_1155249290`（验证时改为`/joint_angles_command`）
- 在终端打印每个t_i时刻的6×1关节向量，格式：`t_i=0: [angle1, angle2, angle3, angle4, angle5, angle6]`

**轨迹保存**:
- 使用提供的`save_trajectory_to_file`函数
- 保存位置：工作空间`src`目录
- 格式：每行一个6×1关节向量，用逗号分隔
- 格式示例：`angle1,angle2,angle3,angle4,angle5,angle6`
- 每个时间步t_i对应一行
- 文件名包含时间戳：`trajectory_YYYYMMDD_HHMMSS.txt`
- 文件在所有轨迹点发布完成后保存
