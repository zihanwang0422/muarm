## 1. Difference between Analytical IK and Numerical IK

### 1.1 Conceptual Differences

**Analytical IK (Analytical Inverse Kinematics)**:
- Directly solves joint angles through geometric analysis and algebraic methods
- Derives closed-form solutions based on the robot's geometric structure
- For 6-DOF robotic arms, complete analytical solutions may be very complex or non-existent
- This implementation adopts a simplified geometric approach based on the structural characteristics of the robotic arm

**Numerical IK (Numerical Inverse Kinematics)**:
- Uses iterative optimization methods to gradually approach the target pose
- Iteratively updates based on the inverse or pseudo-inverse of the Jacobian matrix
- Applicable to various complex robot structures
- This implementation uses Newton-Raphson method with Jacobian pseudo-inverse

### 1.2 Practical Differences

**Formulation**:
- **Analytical IK**: Direct calculation, no iteration required
  - Joint 1: Calculate base rotation angle through projection of target position on xy-plane
  - Joints 2-3: Use cosine theorem and 2D inverse kinematics to solve shoulder and elbow angles
  - Joints 4-6: Extract X-Y-X Euler angle sequence from rotation matrix
  
- **Numerical IK**: Iterative solution
  - Calculate error between current pose and target pose
  - Use Jacobian matrix to compute joint angle updates
  - Repeat iteration until convergence

**Computational Efficiency**:
- **Analytical IK**: 
  - Advantages: Fast computation, results obtained in a single calculation
  - Disadvantages: For complex structures, analytical solutions may not exist or require extensive preprocessing
  
- **Numerical IK**:
  - Advantages: High generality, applicable to various structures
  - Disadvantages: Requires multiple iterations, longer computation time (this implementation uses up to 100 iterations)

**Convergence Behavior**:
- **Analytical IK**: 
  - Always gives a solution (if it exists)
  - However, it may not be optimal and may fail for certain poses
  
- **Numerical IK**:
  - Convergence depends on initial guess and step size
  - This implementation uses damped least squares (Levenberg-Marquardt style) to improve stability
  - Tolerance set to 1e-4, maximum iterations 100

**Suitability for Different Robot Types**:
- **Analytical IK**: 
  - Most suitable for robots with special geometric structures (e.g., 6-DOF robotic arms when analytical solutions exist)
  - For redundant robots, may need to select specific configurations
  
- **Numerical IK**:
  - Applicable to all types of robots
  - Particularly suitable for high-DOF or complex-structured robots
  - Can handle redundancy issues

## 2. Implementation Description

### 2.1 IK Solver Design

#### 2.1.1 Analytical IK Implementation

**Key Geometric Reasoning**:
1. **Joint 1 (Base Rotation)**: 
   - Calculate projection of target position on xy-plane
   - Use `atan2(y, x)` to calculate base rotation angle

2. **Joints 2-3 (Shoulder and Elbow)**:
   - Subtract end-effector offset from target position to obtain wrist position
   - Use cosine theorem to calculate joint 3 angle
   - Calculate joint 2 angle based on joint 3 angle

3. **Joints 4-6 (Wrist)**:
   - Calculate cumulative transformation matrix of first three joints
   - Extract rotation matrix required for wrist
   - Extract X-Y-X Euler angle sequence from rotation matrix (corresponding to rotation axes of joints 4, 5, 6)

**Transformation Chain**:
- Joint rotation axis sequence: Z, X, X, X, Y, X
- End-effector offset applied at the end

#### 2.1.2 Numerical IK Implementation

**Stopping Criteria**:
- Error tolerance: 1e-4 (L2 norm of position and orientation errors)
- Maximum iterations: 100
- Early termination if error is less than tolerance

**Step Size**:
- Fixed step size: 1.0
- Use damping factor λ=0.1 for regularization to improve numerical stability

**Error Tolerance**:
- Position error: Difference between target position and current position (mm)
- Orientation error: Rotation error calculated using axis-angle representation (rad)
- Total error: L2 norm of concatenated position and orientation error vector

**Jacobian Calculation**:
- Use numerical differentiation method
- Apply small perturbation (δ=1e-6) to each joint angle
- Calculate rate of change of position and orientation

**Update Rule**:
- Use damped least squares: `J_pinv = J^T (J J^T + λI)^(-1)`
- Use Moore-Penrose pseudo-inverse if matrix is singular
- Joint angle update: `θ_new = θ_old + step_size * J_pinv * error`

### 2.2 Cubic Trajectory Generation

#### 2.2.1 Task Space Trajectory Generation

**Cubic Polynomial Trajectory**:
For position and Euler angles, use cubic polynomial:
```
p(t) = a₀ + a₁t + a₂t² + a₃t³
```

**Boundary Conditions**:
- Position: `p(0) = start_pos`, `p(T) = end_pos`, `p'(0) = v_start`, `p'(T) = v_end`
- Euler angles: `θ(0) = start_euler`, `θ(T) = end_euler`, `θ'(0) = w_start`, `θ'(T) = w_end`

**Coefficient Calculation**:
```
a₀ = start_value
a₁ = start_velocity
a₂ = (3(end - start) / T²) - (2v_start / T) - (v_end / T)
a₃ = (2(start - end) / T³) + ((v_start + v_end) / T²)
```

**Euler Angle Handling**:
- Use intrinsic Euler angles (XYZ sequence)
- Implement shortest path unwrap to avoid ±π jumps
- Convert Euler angles to rotation matrix, then to quaternion for IK solving

**Rotation Matrix Conversion**:
- Intrinsic XYZ Euler angles: `R = R_z(yaw) * R_y(pitch) * R_x(roll)`
- Rotation matrix to quaternion: Use standard conversion algorithm

#### 2.2.2 Joint Space Trajectory Conversion

**Conversion Process**:
1. For each point in task space trajectory (position + quaternion)
2. Use numerical IK solver to calculate corresponding joint angles
3. Use result from previous trajectory point as initial guess to improve convergence speed and accuracy
4. Store all joint angles as 6×N matrix (N is number of trajectory points)

**IK Solver Selection**:
- Use numerical IK instead of analytical IK because:
  - Numerical IK is more reliable with better convergence
  - Can provide reasonable solutions for every point in the trajectory
  - Analytical IK may fail for certain poses

**Optimization Strategy**:
- Utilize trajectory continuity: Use joint angles from previous point as initial guess for next point
- This significantly improves convergence speed because adjacent trajectory points usually have small changes
- For a trajectory with 1000 points, each point requires only 10-30 iterations on average to converge
