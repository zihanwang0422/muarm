## 1. Why Create Multiple Target Configurations per Reference Joint Angle?

Creating multiple target configurations (8 perturbations) for each reference joint angle set serves several critical purposes:

- **Data augmentation**: Increases dataset size from 5,000 to 40,000 samples without requiring additional FK computations for entirely new poses
- **Local workspace coverage**: By perturbing ±2° around each reference, we ensure dense sampling in the neighborhood of each configuration, helping the network learn smooth mappings
- **Robustness**: The network learns to handle small variations in target poses, which is essential for real-world robot control where slight deviations are common
- **Generalization**: Multiple nearby targets help the model interpolate better between training samples rather than memorizing specific configurations

## 2. Why Normalize Input Features?

Normalization is essential for neural network training:

**Why normalize:**
- **Gradient flow**: Features with different scales (e.g., joint angles in radians [-2, 2] vs positions in cm [-100, 100]) cause unbalanced gradients, making optimization unstable
- **Faster convergence**: Standardized features (mean=0, std=1) allow the optimizer to take consistent step sizes across all dimensions
- **Numerical stability**: Prevents overflow/underflow in activation functions and weight updates

**Without normalization:**
- Training would be extremely slow or fail to converge
- The optimizer would spend most iterations adjusting for scale differences
- Features with larger magnitudes would dominate the loss function
- Risk of vanishing/exploding gradients would increase significantly

## 3. Why Split Data into Train/Test Sets?

The train/test split (80/20 in our case) serves distinct purposes:

**Training set (80%):**
- Used to compute loss and update model weights via backpropagation
- The model directly learns patterns from this data
- Size: 32,000 samples

**Test/Validation set (20%):**
- Used to evaluate model performance on **unseen data**
- Monitors generalization ability during training (via `val_loss` and `val_mae`)
- Triggers early stopping when validation loss stops improving
- Provides unbiased estimate of real-world performance
- Size: 8,000 samples

This separation prevents us from falsely believing the model generalizes when it's merely memorizing training data.

## 4. Why Compute Normalization Scalers from Training Data Only?

This is a critical principle to prevent **data leakage**:

**Reason:**
- The test set represents "future" or "unseen" data that the model has never encountered
- If we include test data statistics (mean/std) in normalization, we're indirectly giving the model information about the test set
- In real deployment, we won't have access to future data statistics—only training statistics

**Correct approach:**
1. Compute `scaler.fit()` on training data only
2. Apply `scaler.transform()` to both train and test sets
3. Save scalers for inference on new robot trajectories

This ensures our validation metrics accurately reflect real-world performance.

## 6. Network Architecture Design

**Architecture:**
```
Input Layer:        13 features (6 reference joints + 4 quaternions + 3 positions)
Hidden Layer 1:     128 neurons, ReLU activation
Hidden Layer 2:     96 neurons, tanh activation  
Hidden Layer 3:     48 neurons, ReLU activation
Output Layer:       6 neurons, linear activation (target joint angles)
Total Parameters:   19,126 (<20,000 requirement ✓)
```

**Design rationale:**

- **Funnel architecture** (128→96→48→6): Progressively compresses high-dimensional input space into 6 target angles, extracting hierarchical features
- **First layer (128, ReLU)**: Large capacity to capture complex nonlinear relationships between pose and joint angles; ReLU for efficient gradient flow
- **Second layer (96, tanh)**: Tanh's symmetric output (-1, 1) helps with centered, normalized data; provides different activation pattern for diversity
- **Third layer (48, ReLU)**: Further refinement with ReLU's sparsity (many neurons inactive)
- **Output layer (6, linear)**: No activation constraint—allows prediction of any joint angle value in continuous space

**Why this works:**
- Balanced capacity: enough neurons to learn IK mapping, not so many to overfit
- Mixed activations: ReLU + tanh provides diverse feature representations
- Meets <20k parameter constraint while achieving <2° accuracy

## 7. Activation Functions: ReLU, tanh, and Linear

**ReLU (Rectified Linear Unit):** `f(x) = max(0, x)`
- **Properties**: Zero for negative inputs, linear for positive inputs
- **Advantages**: Computationally efficient, alleviates vanishing gradient, induces sparsity
- **Use in our network**: Hidden layers 1 & 3 for efficient feature extraction

**tanh (Hyperbolic Tangent):** `f(x) = (e^x - e^(-x))/(e^x + e^(-x))`
- **Properties**: Output range [-1, 1], symmetric around zero, smooth
- **Advantages**: Zero-centered outputs (better than sigmoid), stronger gradients than sigmoid
- **Use in our network**: Hidden layer 2 for diversity—provides different activation pattern than ReLU

**Linear:** `f(x) = x`
- **Properties**: No transformation, direct pass-through
- **Advantages**: Allows unrestricted output range
- **Use in our network**: **Output layer (essential!)** because:
  - Joint angles are continuous values, not bounded to [0,1] or [-1,1]
  - Need to predict any angle within joint limits (e.g., -120° to +90°)
  - Non-linear activation would artificially constrain predictions
  - For regression tasks, output layer must be linear to map to full target range

**Why linear in final layer is critical:** If we used ReLU, all negative angles would be clipped to zero; if we used tanh, outputs would be bounded to [-1, 1], preventing accurate angle prediction.

## 8. Why MSE (Mean Squared Error) is Appropriate

**MSE formula:** `MSE = (1/n) Σ(y_pred - y_true)²`

**Why MSE for this regression task:**

1. **Continuous output**: Joint angles are real-valued, not discrete classes
2. **Penalizes large errors**: Squaring amplifies big mistakes—a 5° error is 25× worse than 1°, which is crucial for robot safety
3. **Smooth gradients**: Differentiable everywhere, providing stable gradient descent
4. **Physical interpretation**: In robotics, squared error relates to energy/torque—larger deviations require exponentially more correction
5. **Statistical optimality**: MSE estimates the conditional mean, giving the expected joint angle for a given pose

**MAE (Mean Absolute Error) as metric:**
- Used alongside MSE for interpretability
- MAE = 0.026 rad directly tells us average prediction error is 1.49°
- MSE focuses on worst cases, MAE focuses on typical cases—both valuable
