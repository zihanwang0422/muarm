## Environment Setup (uv)

This project uses [uv](https://docs.astral.sh/uv/) for environment management. `uv.toml` is pre-configured with the Tsinghua mirror for faster downloads in China.

### 1. Install uv

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 2. Create virtual environment and install dependencies

```bash
cd manipulation_mujoco

# Create .venv with Python 3.10+ and install all dependencies
uv sync
```

> **Note:** Pinocchio (`pin`) requires pre-built wheels. If `uv sync` fails on `pin`, install it first:
> ```bash
> uv pip install pin --extra-index-url https://gepetto.gitlab.io/simple/
> uv sync
> ```

### 3. Activate the environment

```bash
source .venv/bin/activate
```

Or run scripts directly without activating:

```bash
uv run python examples/demo_kinematics.py
```

### 4. Add / remove packages

```bash
uv add opencv-python           # add a dependency
uv add --dev tensorboard       # add a dev-only dependency
uv remove some-package         # remove
uv sync                        # re-sync after editing pyproject.toml manually
```

## src — Core Simulation

The `src` module provides the foundational building blocks. `MuJoCoViewer` is the central abstraction — subclass it and override two hooks to implement any custom simulation.

### MuJoCoViewer

```python
from src import MuJoCoViewer
import numpy as np

class MyEnv(MuJoCoViewer):
    def runBefore(self):
        # Called once before the simulation loop
        self.target = np.array([0.5, 0.0, 0.3])

    def runFunc(self):
        # Called every simulation step
        ee_pos = self.get_body_position("hand")
        error = self.target - ee_pos
        self.data.ctrl[:7] = some_controller(error)

env = MyEnv("models/franka_emika_panda/scene.xml", distance=3, azimuth=-45)
env.run_loop()
```

**Open API:**

| Method | Description |
|--------|-------------|
| `run_loop()` | Launch viewer and run the simulation loop |
| `set_timestep(dt)` | Change simulation timestep |
| `get_body_position(name)` | `[x, y, z]` of a named body |
| `get_body_pose_euler(name)` | `[x, y, z, roll, pitch, yaw]` of a named body |
| `get_body_quat(name)` | `[w, x, y, z]` quaternion of a named body |
| `set_mocap_position(name, pos)` | Move a mocap target body |
| `set_mocap_quat(name, euler)` | Rotate a mocap target body |
| `get_geom_position(name)` | World position of a geom |
| `set_geom_position(name, pos)` | Teleport a geom |
| `get_contact_info()` | List of all active contact pairs |
| `add_visual_geom(...)` | Overlay visual markers in the viewer |
| `add_obstacles(...)` | Inject collision obstacles into the model |
| `set_control(ctrl)` | Set actuator control signals |
| `get_joint_positions(n)` | First `n` joint positions |
| `get_joint_velocities(n)` | First `n` joint velocities |

### PIDController

```python
from src import PIDController
import numpy as np

pid = PIDController(kp=100.0, ki=0.0, kd=10.0, n_dims=7)

# In the control loop:
torque = pid.compute(q_desired - q_current, dt=0.001)  # → shape (7,)
pid.reset()  # clear integrator
```

### MPCController

```python
from src import MPCController
import numpy as np

mpc = MPCController(
    n_joints=7,
    horizon=10,
    dt=0.002,
    q_weight=10.0,    # position tracking weight
    u_weight=0.01,    # control effort weight
    du_weight=0.1,    # smoothness weight
    u_max=np.full(7, 10.0),
)

# In the control loop (q_ref is a (T, 7) reference trajectory):
u = mpc.compute(q_current, q_dot_current, q_ref)  # → shape (7,)
```

## kinematic — Kinematics & Trajectory

### PandaKinematics

Pinocchio + CasADi/IPOPT optimization-based IK with joint limit constraints.

```python
from kinematic import PandaKinematics
from utils import transform2mat
import numpy as np

kin = PandaKinematics(ee_frame="link7")
kin.build_from_mjcf("models/franka_emika_panda/panda.xml")
# or: kin.build_from_urdf("path/to/panda_arm.urdf")

# Forward kinematics → 4x4 transform
q = np.array([0, -np.pi/4, 0, -3*np.pi/4, 0, np.pi/2, np.pi/4])
T = kin.fk(q)
pos = kin.get_ee_position(q)       # [x, y, z]
pose = kin.get_ee_pose(q)          # (x, y, z, roll, pitch, yaw)

# Inverse kinematics
target = transform2mat(0.5, 0.0, 0.3, np.pi, 0, 0)
q_sol, info = kin.ik(target)
print(info["success"])             # True / False
print(info["sol_tauff"])           # feedforward torques

# Jacobian: shape (6, N)
J = kin.jacobian(q)
```

### TrajectoryGenerator

```python
from kinematic import TrajectoryGenerator
import numpy as np

tg = TrajectoryGenerator()

# Cubic polynomial — zero-velocity at start and end
traj = tg.cubic(q_start, q_end, duration=3.0, dt=0.002)
# traj["positions"]  → (T, 7)
# traj["velocities"] → (T, 7)
# traj["times"]      → (T,)

# Multi-waypoint cubic
traj = tg.multi_waypoint([q0, q1, q2, q0], segment_duration=2.0, dt=0.002)

# Linear interpolation
pts = tg.linear(q_start, q_end, steps=200)     # → (200, 7)

# Cartesian arc
arc = tg.cartesian_arc(center=[0.4, 0, 0.4], radius=0.15,
                        start_angle=0, end_angle=np.pi,
                        axis='z', steps=100)    # → (100, 3)
```

## control — Robot Controllers

### ImpedanceController

**Joint-space** — τ = Kp · (q_des − q) − Kd · q̇

```python
from control import ImpedanceController

ctrl = ImpedanceController(
    n_joints=7,
    kp=[600, 600, 600, 600, 250, 150, 50],
    kd=[50,  50,  50,  50,  20,  20, 10],
    mode='joint',
)
tau = ctrl.compute_joint(q_desired, q_current, q_dot_current)
data.ctrl[:7] = tau
```

**Cartesian-space** — τ = Jᵀ · (−Kp · x_err − Kd · ẋ)

```python
ctrl = ImpedanceController(n_joints=7, kp=200.0, kd=20.0, mode='cartesian')

tau = ctrl.compute_cartesian(
    pos_desired=target_pos,        # [x, y, z]
    pos_current=ee_pos,
    vel_current=ee_vel_6d,         # [vx, vy, vz, wx, wy, wz]
    jacobian=J,                    # (6, 7)
    R_desired=R_des,               # (3, 3) optional
    R_current=R_cur,
)
```

### AdmittanceController

M·ẍ + B·ẋ + K·(x − x_ref) = F_ext

The robot tracks a modified desired pose that "gives way" to external forces.

```python
from control import AdmittanceController
from utils import transform2mat

ctrl = AdmittanceController(
    M=10.0,   # virtual inertia — lighter = more compliant
    B=50.0,   # virtual damping
    K=100.0,  # virtual stiffness
)
ctrl.set_reference(initial_ee_pose_6d)   # equilibrium point

# In the control loop:
F_ext = np.array([0, 0, -5, 0, 0, 0])   # 5 N downward
desired_pose = ctrl.update(F_ext, current_pose, current_vel, dt=0.001)
T = ctrl.get_transform(desired_pose)     # → 4x4 matrix for IK
```

## learning — Reinforcement Learning

All environments implement the Gymnasium interface and support SubprocVecEnv parallel training.

### PandaReachEnv

Agent moves the Panda end-effector to a randomly sampled 3D target.

| | |
|---|---|
| **Observation** | `joint_pos(7) + goal_pos(3)` — dim 10 |
| **Action** | `[-1, 1]^7` scaled to joint position ranges |
| **Done** | Goal reached (dist < 0.05 m) or timeout (20 s) |

### PandaObstacleAvoidEnv

Reach a goal while avoiding a randomized spherical obstacle.

| | |
|---|---|
| **Observation** | `joint_pos(7) + goal_pos(3) + obs_pos(3) + obs_radius(1)` — dim 14 |
| **Done** | Goal reached, collision, or timeout |

### Training & Testing

```python
from learning import PandaReachEnv, PandaObstacleAvoidEnv, train_ppo, test_ppo

# Train
train_ppo(
    env_class=PandaReachEnv,
    n_envs=16,
    total_timesteps=5_000_000,
    model_save_path="panda_ppo_reach",
)

# Test
test_ppo(
    env_class=PandaReachEnv,
    model_path="panda_ppo_reach",
    total_episodes=10,
)
```

Custom network architecture:

```python
import torch.nn as nn

train_ppo(
    env_class=PandaObstacleAvoidEnv,
    n_envs=32,
    total_timesteps=20_000_000,
    policy_kwargs=dict(
        activation_fn=nn.LeakyReLU,
        net_arch=[dict(pi=[512, 256, 128], vf=[512, 256, 128])],
    ),
    learning_rate=lambda f: 1e-4 * (1 - f),  # linear decay
)
```

Command line:

```bash
uv run python examples/demo_ppo_reach.py train
uv run python examples/demo_ppo_reach.py test
```

## grasp — Vision-based Grasping

### SimCamera

Captures images from cameras defined in the MJCF scene.

```python
from grasp import SimCamera
import mujoco

model = mujoco.MjModel.from_xml_path("models/franka_emika_panda/scene_pos.xml")
data  = mujoco.MjData(model)
cam   = SimCamera(model, data)

# scene_pos.xml includes: "overhead_cam" and "wrist_cam"
rgb   = cam.capture_fixed("overhead_cam", width=640, height=480)  # (H,W,3) uint8
depth = cam.capture_depth("overhead_cam", width=640, height=480)  # (H,W) float32
depth_m = cam.depth_to_meters(depth)                              # metres

# Tracking camera that follows a body
rgb = cam.capture_tracking("hand", distance=0.5, elevation=-30)
```

### GraspPipeline

A MuJoCoViewer subclass that runs a full detect → approach → descend → close gripper → lift cycle.

```python
from grasp import GraspPipeline

pipeline = GraspPipeline(
    scene_xml="models/franka_emika_panda/scene_pos.xml",
    camera_name="overhead_cam",   # set None to use simulation-state detection
)
pipeline.run_loop()
```

Override `detect_object()` to swap in a real perception model:

```python
class MyGrasp(GraspPipeline):
    def detect_object(self, rgb_image=None):
        # Use GraspNet, AnyGrasp, or any other model
        return your_detector.predict(rgb_image)   # returns [x, y, z]
```

## utils — Math Utilities

Quaternion convention: `[w, x, y, z]`.

```python
from utils import (
    quat2rotmat,   # quat              → (3,3) rotation matrix
    rotmat2quat,   # (3,3)             → quat
    euler2rotmat,  # (roll,pitch,yaw)  → (3,3)  [ZYX order]
    euler2quat,    # (roll,pitch,yaw)  → quat
    quat2euler,    # quat              → (roll, pitch, yaw)
    transform2mat, # (x,y,z,r,p,y)    → (4,4) homogeneous matrix
    mat2transform, # (4,4)             → (x,y,z,roll,pitch,yaw)
    damped_pinv,   # J†= Jᵀ(JJᵀ + λ²I)⁻¹
)
import numpy as np

T    = transform2mat(0.5, 0.0, 0.3, np.pi, 0, 0)
pose = mat2transform(T)             # (x, y, z, roll, pitch, yaw)

J      = np.random.randn(6, 7)
J_pinv = damped_pinv(J, lambda_d=0.05)   # (7, 6)
```
